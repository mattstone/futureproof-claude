EPM Overview

Model structure and stakeholder relationships

FutureProof's Equity Partnering Model (EPM) is a fund-based income solution that transforms home equity into sustainable monthly income. The model connects four stakeholders through a sophisticated distribution system powered by monthly rebalancing and performance monitoring.

The Four Stakeholders

Stakeholder Role Capital Flow
Customer Homeowner who partners equity for monthly income Receives monthly distributions from fund returns
Investment Partner Manages invested capital (S&P 500 ETFs) Generates daily returns; manages fund performance and rebalancing
Lender Provides mortgage product & customer relationship Receives margin on monthly distributions (not traditional interest)
Wholesale Funder Supplies capital to lenders via funding lines Receives wholesale margin on deployed capital

Monthly Distribution Flow

Daily Fund Returns: Investment Partner's ETF holdings generate daily returns (on average 10% p.a. over 30 years, tracked to S&P 500).

Monthly Distribution: System aggregates daily returns and calculates monthly payout amounts based on:

  • Customer's equity percentage (typically 2% of home value)
  • Fund performance month-to-date
  • Interest holiday status (if triggered)

Amount to Customer: Customer receives their share of monthly returns as income.

Lender Margin: Lender deducts their margin from distribution (0.75% to 2% depending on tier).

Wholesale Margin: Wholesale Funder receives their cut (0.2% to 3.6% depending on structure).

FutureProof Margin: Platform takes its cut (0.25%) for ongoing management.

System Architecture

Platform components and core services

Core Platform

Rails 8.1 + Hotwire + Stimulus: Single monolithic Rails application with real-time updates via Hotwire (Turbo) for seamless user interactions and Stimulus controllers for client-side interactivity.

PostgreSQL Database: Single source of truth for all customer, transaction, and account data with full audit trails.

Kafka Event Queue: Asynchronous event streaming for agent coordination and system-wide notifications.

Redis Cache: Session management, rate limiting, and temporary computation caching.

S3 Storage: Document archival, statements, and audit logs.

Core Services

Distribution Engine: Calculates monthly distributions based on fund returns, applies margins, and initiates payments.

Compliance Engine: Enforces regulatory rules, tracks audit trail, and validates all transactions.

Fraud Detection: Real-time pattern matching against known fraud scenarios and automated alerts.

Interest Holiday Logic: Monitors fund performance and triggers automated holidays when thresholds breached. FutureProof makes all holiday decisions (entry/exit). Customers are not notified. During holiday, customer income continues; lender margin is accrued to loan term instead.

Notification Service: Sends customer statements, alerts, and lender/funder reports via email and SMS.

Document Engine: Generates contracts, disclosures, and monthly statements.

Agent Communication Flow

All agents publish events to Kafka; others subscribe to relevant events and react asynchronously. This decouples the four workflows and enables real-time coordination without tight coupling.

Deal Agent publishes deal.created when customer onboarded. Distribution Agent and Compliance Agent subscribe to initiate workflows.

Distribution Agent publishes distribution.monthly on cutoff date. Lender and Wholesale Funder receive payments.

Investment Agent publishes holiday.triggered or holiday.released. Distribution Agent adjusts payment calculations.

Investment Agent publishes holiday.triggered or holiday.released when fund thresholds breached. FutureProof makes decision; customer is unaware.

Compliance Agent publishes fraud.detected or rule.violation. System escalates to lender/admin for review.

Interest Holiday Mechanism

Dynamic payment suspension based on fund performance

How Holidays Work

FutureProof Decision Authority: FutureProof (not the lender) determines when a customer enters and exits an interest holiday. This is an automated decision based on rolling fund performance thresholds. Customers are unaware holidays are active—there is no notification.

How It Works: During a holiday, the customer continues to receive their full monthly income as normal. However, the accrued interest (lender's margin) is added to the loan term rather than paid out. This extends the loan maturity date automatically.

Parameter Value Meaning
Entry Threshold 0.90 (90%) If fund value drops to 90% of starting value, entry conditions met
Exit Threshold 1.458 (145.8%) When fund recovers to 145.8% of original value, holiday automatically released
Maximum Duration Variable Holiday automatically expires if conditions not met within timeframe
Effect Loan extension Accrued interest added to loan term, extending maturity date

Timeline Example

Months 1–10: Normal Operations

Fund performing well. Customer receives monthly distributions. FutureProof monitors thresholds daily.

Month 11: Market Downturn

Fund value drops to 88% of starting value. Entry threshold (90%) breached. FutureProof automatically triggers holiday (customer unaware).

Months 12–18: Holiday Active

Customer still receives full monthly income. Lender's margin is NOT paid out but instead accrued and added to loan principal. Loan maturity extends. Customer has no notification of holiday status.

Month 19: Recovery Begins

Fund recovers to 145.8% threshold. FutureProof automatically releases holiday. Accrued interest locked in; loan term now 31 years instead of 30. Normal margin payments resume.

Compliance & Fraud Detection

Regulatory adherence and system safeguards

Compliance Rules

Rule Validation Failure Action
KYC/AML Verification Customer identity verified before onboarding Application rejected; customer notified
Responsible Lending Fund amount appropriate for customer profile Risk tier adjusted; margin increased
Distribution Accuracy Monthly calculations verified; margins correct Distribution held pending audit; customer notified
Fund Performance Audit Daily returns match investment partner reports Discrepancy flagged; reconciliation initiated
Holiday Logic Thresholds correctly applied; payment suspension verified Holiday manually reviewed; investor notified
Data Privacy PII access logged; encryption verified Unauthorized access alert; incident investigation

Real-Time Fraud Monitoring

Account Access Anomalies: Logins from unusual locations, rapid device changes, after-hours access patterns.

Unusual Transaction Patterns: Multiple withdrawals same day, requests immediately after distribution, systematic holiday manipulation.

Data Inconsistencies: Mismatches between reported and actual fund values, distribution calculation errors, lender margin discrepancies.

Regulatory Violations: KYC/AML failures, undisclosed conflicts of interest, unauthorized fund transfers.

Alerts are automatically escalated to lenders and funders. Critical incidents require manual review.

Audit & Reporting

Audit Trail: Every transaction, margin payment, and holiday trigger is recorded with timestamp, user ID, and change details. Logs retained for 7 years minimum (compliance requirement).

Monthly Reporting: Lenders and wholesale funders receive automated reports on portfolio health, margin collections, and default scenarios.

Annual Compliance Certification: To be performed by external auditor. Covers all regulatory requirements across Australia, UK, and US jurisdictions.

Customer Journey

Three phases from origination to exit

Three Phases

PHASE 1: ORIGINATION (Weeks 1–4)

Customer applies: Provides property details, equity percentage, income expectations through portal.

Deal Agent processes: KYC/AML checks, property valuation, risk tier assignment, document generation.

e-Signature: Customer reviews and signs loan documents, investment agreement.

Funding confirmation: Lender & Wholesale Funder approve. Capital transferred. Account activated.

PHASE 2: ONGOING SERVICING (Months 1–30+)

Monthly distributions: Distribution Agent calculates returns every month. Customer receives payment on fixed date.

Performance monitoring: Investment Agent tracks fund daily. Generates rolling metrics visible on customer dashboard.

Interest holidays: If performance drops, holiday automatically triggered. Customer notified of status change.

Statements & reports: Monthly statements show distribution amount, margins paid, fund performance. Accessible via portal.

Account management: Customer can request early exit, modify settings. Lender reviews requests within 5 business days.

PHASE 3: EXIT (Early or Maturity)

Customer-initiated: Request early repayment. Deal Agent calculates exit position (fund value minus outstanding fees).

Lender approval: Lender reviews exit request, confirms repayment terms.

Settlement: Distribution Agent processes final payment. All accounts reconciled. Loan closed.

Documents archived: All transaction history, statements, audit logs preserved for 7 years. Compliance Agent completes final audit report.

Architecture Notes

Monolithic Rails App: Single codebase simplifies deployment, testing, and debugging. Real-time features powered by Hotwire (Turbo Streams) enable dashboard updates without JavaScript complexity.

Event-Driven Agents: Agents implemented as background jobs (Sidekiq/Resque) listening to Kafka topics. Decouples origination, distribution, monitoring, and compliance workflows.

PostgreSQL as Single Source of Truth: All customer, transaction, and account data stored in single database. Full audit trail via database triggers.

Deployment & Quality Assurance

Release process and testing strategy

Testing Strategy

Unit Tests: Core business logic (distribution calculations, holiday logic, fraud detection rules).

Integration Tests: Agent workflows, end-to-end customer journeys, payment processing.

Performance Tests: Monthly distribution batch processing at scale (1000s of customers), query optimization.

Security Tests: Authentication, authorization, data access controls, PII encryption.

Manual Testing: Customer portal UX, lender reporting accuracy, admin workflows.

Continuous Deployment

GitHub Actions: Automated tests run on every PR. Tests must pass before merge to main.

Staging Environment: Deployed automatically from main branch. Manual smoke tests before production release.

Production Release: Blue-green deployment on Fly.io. Automatic rollback if health checks fail within 5 minutes.

Monitoring & Alerting

Sentry: Real-time error tracking. Critical errors trigger PagerDuty alerts.

CloudWatch Logs: Full transaction logs, agent activity, performance metrics.

Dashboard: Real-time visibility into system health: agent status, transaction volume, error rates, margin collections.
