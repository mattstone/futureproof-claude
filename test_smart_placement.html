Smart Placement Rules

Main Chain Nodes: Go directly under trigger with 140px vertical spacing
First Node: Always positioned directly below trigger (no horizontal offset)
Condition Branches: NO branch 300px left, YES branch 300px right with slight vertical offset
Overflow Handling: If position occupied, try below → right → left in order
Node Reference: All positioning uses trigger node as the anchor point
Quick Actions: Use same smart positioning rules for consistency

Benefits of Smart Placement

  • Predictable, organized workflow structure
  • Easy to follow main execution path
  • Clear visual separation of conditional branches
  • Consistent spacing and alignment
  • No more scattered, chaotic node placement
  • Professional workflow appearance
  • Scales well with complex workflows
